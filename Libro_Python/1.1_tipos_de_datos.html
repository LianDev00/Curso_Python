int
Enteros
tipo · int · números sin decimales

Un int representa cualquier número entero — positivo, negativo o cero. No tiene límite de tamaño en Python (a diferencia de otros lenguajes). Se usa para contar, indexar, rangos, y cualquier cálculo que no necesite decimales.

Crear enteros
edad       = 25
temperatura = -8
cero       = 0
millon     = 1_000_000     # el guión bajo es separador visual, válido en Python
binario    = 0b1010        # int en base 2 → 10
hexa       = 0xFF          # int en hexadecimal → 255

print(type(edad))            # <class 'int'>
A diferencia de otros lenguajes: Python no tiene int32, int64, long, etc. Un solo tipo int que crece sin límite. 10 ** 1000 funciona perfectamente.
Método / Función Descripción Ejemplo → resultado
abs(n) Valor absoluto abs(-7) → 7
pow(base, exp) Potencia (igual que **) pow(2, 8) → 256
divmod(a, b) Retorna (cociente, resto) a la vez divmod(17, 5) → (3, 2)
bin(n) Convierte a string binario bin(10) → '0b1010'
hex(n) Convierte a string hexadecimal hex(255) → '0xff'
oct(n) Convierte a string octal oct(8) → '0o10'
n.bit_length() Cuántos bits necesita para representarse (10).bit_length() → 4
Usos típicos
# contar
intentos = 0
intentos += 1

# índice de lista
lista[0]
lista[-1]

# rango
for i in range(0, 10, 2):    # 0 2 4 6 8
    print(i)

# módulo (resto) — muy usado para par/impar o ciclos
7 % 2    # → 1  (impar)
8 % 2    # → 0  (par)
10 % 3   # → 1  (el resto de dividir 10 entre 3)

# división entera — retorna int, descarta decimales
17 // 5  # → 3   (no 3.4)

Decimales
tipo · float · números con punto decimal

Un float representa números con parte decimal. Internamente Python los almacena en formato IEEE 754 de 64 bits — esto tiene consecuencias importantes en precisión que conviene conocer desde el principio.

Crear floats
precio   = 19.99
pi       = 3.14159
negativo = -0.5
grande   = 1.5e6      # notación científica → 1500000.0
chico    = 2.3e-4     # → 0.00023
forzado  = 5.         # el punto solo ya lo hace float → 5.0

print(type(precio))       # <class 'float'>
El problema de precisión del float: 0.1 + 0.2 en Python devuelve 0.30000000000000004, no 0.3. Esto no es un bug — es cómo funciona el estándar IEEE 754. Para dinero o comparaciones exactas, usa el módulo decimal.
Precisión y redondeo
# el problema
0.1 + 0.2              # → 0.30000000000000004

# round() — redondea a N decimales
round(3.14159, 2)     # → 3.14
round(2.675, 2)       # → 2.67  (sorpresa: IEEE 754)
round(4.5)            # → 4  (redondeo bancario: al par más cercano)
round(5.5)            # → 6

# comparar floats correctamente — nunca con ==
import math
math.isclose(0.1 + 0.2, 0.3)     # → True

# formato de salida con f-string
precio = 19.9999
print(f"S/ {precio:.2f}")         # → S/ 20.00
print(f"{precio:.4f}")            # → 19.9999
print(f"{precio:10.2f}")          # → '     20.00'  (ancho 10)
FunciónDescripciónEjemplo → resultado
round(n, d) Redondea a d decimales round(3.567, 2) → 3.57
abs(n) Valor absoluto abs(-3.14) → 3.14
math.floor(n) Redondea hacia abajo → int math.floor(3.9) → 3
math.ceil(n) Redondea hacia arriba → int math.ceil(3.1) → 4
math.sqrt(n) Raíz cuadrada math.sqrt(16) → 4.0
math.isclose(a, b) Compara floats con tolerancia math.isclose(0.1+0.2, 0.3) → True
n.is_integer() True si el float no tiene parte decimal real (4.0).is_integer() → True
float('inf') Infinito positivo (válido en Python) float('inf') > 10**100 → True
Usos típicos
# precios y porcentajes
precio    = 150.0
descuento = precio * 0.18           # 18% de descuento
total     = precio - descuento
print(f"Total: S/ {total:.2f}")

# IMC
peso   = 70.0
altura = 1.75
imc    = peso / (altura ** 2)
print(f"IMC: {imc:.2f}")

# promedio — división siempre da float
notas    = [15, 18, 12]
promedio = sum(notas) / len(notas)   # → 15.0 (float)
bool
Booleanos
tipo · bool · True / False

Un bool solo puede ser True o False. En Python, bool es subclase de int — True vale 1 y False vale 0. Esto tiene implicaciones útiles: puedes sumar booleanos para contar cuántos son verdaderos.

Crear y usar booleanos
activo   = True
enviado  = False

# resultado de una comparación → siempre bool
5 > 3          # → True
5 == 3         # → False
10 != 10       # → False
7 >= 7         # → True

# bool es subclase de int
True + True    # → 2
True * 5      # → 5
False + 1     # → 1

# contar cuántos son verdaderos en una lista
aprobados = [True, False, True, True]
sum(aprobados)     # → 3

Valores truthy y falsy — en Python cualquier objeto puede evaluarse como bool. No solo True/False:

Falsy — se evalúan como False
False
0          # int cero
0.0        # float cero
""         # string vacío
[]         # lista vacía
{}         # dict vacío
None       # el tipo None
Truthy — se evalúan como True
True
1          # cualquier int ≠ 0
-3.14      # cualquier float ≠ 0.0
"hola"     # string no vacío
[1,2,3]    # lista no vacía
{"a":1}    # dict no vacío
Operadores lógicos: and · or · not
# and — ambos deben ser True
True and True     # → True
True and False    # → False

# or — al menos uno True
False or True     # → True
False or False    # → False

# not — invierte
not True          # → False
not False         # → True

# uso real: validar múltiples condiciones
edad   = 20
dinero = 150
if edad >= 18 and dinero >= 100:
    print("Puede comprar")

# uso de truthy — si la lista no está vacía
nombres = ["Ana", "Luis"]
if nombres:               # equivale a: if len(nombres) > 0
    print("Hay nombres")

# bool() — convertir cualquier valor a bool explícitamente
bool(0)       # → False
bool(42)      # → True
bool("")      # → False
bool("hola") # → True

None
tipo · NoneType · ausencia de valor

None es el único valor del tipo NoneType. Representa la ausencia de valor — no es cero, no es string vacío, no es False. Es "nada". Lo verás mucho como valor por defecto de funciones que no retornan nada explícitamente, o para indicar que algo no fue encontrado.

Dónde aparece None
# una función sin return devuelve None
def saludar():
    print("Hola")            # imprime, pero no retorna nada

resultado = saludar()       # resultado es None
print(resultado)             # → None

# como señal de "no encontrado"
def buscar(lista, objetivo):
    for item in lista:
        if item == objetivo:
            return item
    return None              # no se encontró

# inicializar una variable sin valor aún
usuario = None
# ... más adelante se asigna si se encuentra

# .get() de diccionario retorna None si la clave no existe
d = {"a": 1}
d.get("z")                  # → None
Comparar None siempre con is, no con ==:
x is None → correcto
x == None → funciona pero no es idiomático — algunos objetos pueden sobrecargar == y dar resultados inesperados.
Verificar None correctamente
valor = None

# correcto
if valor is None:
    print("No hay valor")

if valor is not None:
    print(f"Valor: {valor}")

# None es falsy — también funciona así
if not valor:
    print("Falsy")

# None en type()
type(None)     # → <class 'NoneType'>

Conversión de tipos
int · float · bool · str — casting explícito

Python no convierte tipos automáticamente (salvo en operaciones numéricas mixtas). Para cambiar de tipo usas funciones explícitas: int(), float(), str(), bool(). Esto se llama casting.

Tabla de conversiones posibles
# → int
int("42")          # → 42       ✓ string numérico
int(3.9)           # → 3        ✓ trunca (no redondea)
int(True)          # → 1
int(False)         # → 0
int("3.14")        # ✗ ValueError — no acepta decimales en string
int("hola")        # ✗ ValueError

# → float
float("3.14")      # → 3.14
float(7)           # → 7.0
float(True)        # → 1.0
float("inf")       # → inf  (infinito)

# → str
str(42)            # → "42"
str(3.14)          # → "3.14"
str(True)          # → "True"
str(None)          # → "None"

# → bool
bool(0)            # → False
bool(1)            # → True
bool("")           # → False
bool("cualquier") # → True
input() siempre devuelve str. Si el usuario escribe 25, Python lo recibe como "25". Si intentas hacer aritmética directamente obtendrás un error. Siempre convierte: edad = int(input("Edad: "))
Conversiones mixtas automáticas en operaciones
# int + float → Python convierte el int a float automáticamente
3 + 1.5       # → 4.5  (float)
10 / 2        # → 5.0  (/ siempre devuelve float)
10 // 2       # → 5   (// devuelve int si ambos son int)
10 // 2.0     # → 5.0 (// devuelve float si hay algún float)

# str + str → concatena (no suma)
"3" + "4"     # → "34"   (¡no 7!)
# str + int → TypeError — Python no convierte automáticamente
"edad: " + 25 # ✗ TypeError
"edad: " + str(25)  # ✓ → "edad: 25"
type() e isinstance() — verificar el tipo
x = 42

# type() — devuelve el tipo exacto
type(x)                      # → <class 'int'>
type(x) == int               # → True

# isinstance() — verifica tipo incluyendo herencia (más recomendado)
isinstance(x, int)           # → True
isinstance(x, (int, float))  # → True (cualquiera de los dos)
isinstance(True, int)        # → True  (bool hereda de int)

Operadores de asignación compuesta
n = 10

n += 3    # n = n + 3  → 13
n -= 2    # n = n - 2  → 11
n *= 2    # n = n * 2  → 22
n /= 4    # n = n / 4  → 5.5   (siempre float)
n //= 2   # n = n // 2 → 2
n %= 3    # n = n % 3  → 2
n **= 3   # n = n ** 3 → 8
Operadores de comparación — siempre retornan bool
5 == 5     # igualdad        → True
5 != 3     # distinto        → True
7 > 3      # mayor           → True
3 < 7      # menor           → True
5 >= 5     # mayor o igual   → True
4 <= 3     # menor o igual   → False

# Python permite encadenar comparaciones (muy útil)
0 < 5 < 10       # → True  (5 está entre 0 y 10)
18 <= 25 < 65     # → True  (25 es adulto)
# equivale a: 18 <= 25 and 25 < 65 — más legible
Precedencia de operadores (de mayor a menor)
# 1. Paréntesis         ()
# 2. Potencia            **
# 3. Negación unaria     -n
# 4. *, /, //, %
# 5. +, -
# 6. Comparaciones       ==, !=, >, <, >=, <=
# 7. not
# 8. and
# 9. or

# ejemplos
2 + 3 * 4        # → 14   (no 20 — * antes que +)
(2 + 3) * 4      # → 20   (paréntesis primero)
2 ** 3 ** 2      # → 512  (** asocia de derecha a izquierda: 2**(3**2))
-2 ** 2          # → -4   (** antes de negación: -(2**2))
(-2) ** 2        # → 4    (negación dentro del paréntesis)
Regla práctica: cuando tengas duda sobre precedencia, usa paréntesis. No solo resuelven la ambigüedad — hacen el código más legible para cualquiera que lo lea después.
