02
Contador de palabras con diccionario
Intermedio

8Pide al usuario una frase. 1Recorre cada palabra con un for. 7Guárdala en un diccionario donde las claves son las palabras y los valores los conteos. Al final 2muestra las palabras ordenadas de mayor a menor frecuencia.

Pasos — sin código
8
Variable / Input
Pedir la frase con input() y aplicar .lower().strip() para normalizar. Luego separar con .split() para obtener una lista de palabras.
7
Diccionario — inicializar
Crear conteo = {} antes del bucle. Todo acumulador va ANTES del bucle que lo llena.
1
Bucle for
Recorrer cada palabra de la lista con for palabra in palabras. La cantidad es conocida → for, nunca while.
7
Diccionario — patrón get()
Dentro del for: conteo[palabra] = conteo.get(palabra, 0) + 1. No necesitas if para verificar si existe — .get() maneja el caso de clave nueva automáticamente.
2
print — resultados ordenados
Fuera del bucle, ordenar con sorted(conteo.items(), key=lambda x: x[1], reverse=True) y recorrer con for para imprimir cada par.
💡

Clave: el patrón dict.get(clave, 0) + 1 es la forma estándar de contar en Python. Evita tener que escribir if para verificar si la clave existe.

03
Agenda de contactos con diccionario
Avanzado

Crea una agenda donde 7cada contacto tiene nombre, teléfono y email en un diccionario anidado. 3El menú se repite indefinidamente. 5Cada opción es una función separada: agregar, 1listar recorriendo todos los contactos, y 4buscar comparando si el nombre coincide. Usa .get() para 4verificar si existe sin KeyError.

Pasos — sin código
7
Diccionario anidado — estructura
Primero diseña la estructura: agenda = {} donde cada clave es el nombre y el valor es {"tel": ..., "email": ...}. Diseña datos antes de funciones.
5
Funciones def — una por responsabilidad
Identificar las 3 funciones: agregar(agenda), buscar(agenda, nombre), listar(agenda). Cada función hace UNA sola cosa y recibe la agenda como parámetro.
3
while True — menú principal
No sabemos cuántas veces usará el menú → while True. Dentro: mostrar opciones, pedir opción, llamar función. La opción salir ejecuta break.
4
Guardia al inicio de funciones
En listar y buscar: si len(agenda) == 0 → imprimir "Agenda vacía" y return. La guardia sale de la función antes de procesar nada.
7
Agregar y buscar con .get()
Agregar: agenda[nombre] = {"tel": tel, "email": email}. Buscar: contacto = agenda.get(nombre) — si retorna None, no existe. Sin KeyError.
1
for — listar todos
En la función listar: for nombre, datos in agenda.items(). Cantidad conocida → for. Mostrar datos["tel"] y datos["email"].
2
print — confirmar cada acción
Después de agregar: "Contacto agregado". Después de buscar: mostrar datos o "No encontrado". El usuario siempre debe saber qué pasó.
04
Estadísticas de notas con tuplas
Intermedio

Crea una 5función analizar(nombre, notas) que reciba 6una lista de tuplas (nombre, notas_tupla). Calcula 8promedio, 8nota máxima y mínima, y 4si aprobó (promedio ≥ 60). 1Recorre la lista con for usando unpacking y llama a la función por cada alumno.

Pasos — sin código
6
Lista de tuplas — estructura
Definir alumnos = [("Ana", (85,90,78)), ("Luis", (70,65,80)), ...] directamente en el código. Lista porque hay varios. Tupla porque las notas son fijas e inmutables.
5
Función def — solo calcula
analizar(nombre, notas) recibe los dos valores. Solo calcula y muestra — no pide input. Esta separación es clave.
8
Variables de cálculo
Dentro de la función: promedio = sum(notas) / len(notas). max(notas) y min(notas) funcionan directamente sobre la tupla sin necesitar un for.
4
if — determinar estado
if promedio >= 60 → "Aprobado". else → "Reprobado". Una sola condición, una sola decisión.
1
for con unpacking
for nombre, notas in alumnos — el unpacking extrae ambos valores de la tupla en una sola línea. Mucho más limpio que alumnos[i][0] y alumnos[i][1].
2
print — reporte
Mostrar todos los datos dentro de la función analizar(). El for solo llama a la función — no imprime directamente.
💡

Unpacking: cuando una tupla tiene dos elementos, puedes asignarlos a dos variables en el mismo for: for a, b in lista_de_tuplas. Es más legible que acceder por índice.

05
Inventario de tienda
Avanzado

Crea un inventario usando un 7diccionario donde cada producto tiene precio y stock. 3El menú se repite hasta que el usuario salga. 5Cada opción es una función: ver inventario, comprar 4validando que haya suficiente stock, y calcular el valor total. 1Al ver inventario, recorre todos los productos y 8actualiza el stock después de cada venta.

Pasos — sin código
7
Diccionario anidado — diseñar primero
inventario = {"manzana": {"precio": 2.5, "stock": 100}, ...}. Diseña la estructura antes de escribir cualquier función. Todo lo demás depende de esto.
5
Funciones — una por opción del menú
Tres funciones: ver_inventario(inv), comprar(inv), valor_total(inv). El while solo llama funciones — nunca hace lógica directamente.
3
while True — menú principal
Menú infinito → while True. Pedir opción, llamar función. La opción "salir" ejecuta break. Sin else adicional.
1
for — ver inventario
En ver_inventario: for nombre, datos in inv.items(). Usar f-strings con alineación para simular tabla: f"{nombre:<15} S/{datos['precio']:>6.2f}".
4
if — dos validaciones en comprar
En comprar: primero if nombre not in inv → "no existe". Luego if cantidad > inv[nombre]["stock"] → "stock insuficiente". Solo si pasa ambas → proceder.
8
Variable — actualizar stock
Después de validar: inv[nombre]["stock"] -= cantidad. El operador -= resta y guarda en la misma línea.
8
Acumular valor total
En valor_total: total = sum(d["precio"] * d["stock"] for d in inv.values()). Generator expression — más eficiente que un for con append.
✅

Regla: cuando el enunciado dice "cada opción es una función", primero lista las opciones y crea una función por cada una. El while solo llama funciones — nunca tiene lógica propia.

07
Buscar en lista con función
Intermedio

Crea una 5función buscar(lista, elemento) que 1recorra la lista con un for y 4retorne True si el elemento está, False si no. No puedes usar el operador in directamente — 4simula lo que hace in con un bucle manual. 2Pruébala con una lista de nombres predefinida.

Pasos — sin código
6
Lista predefinida
Definir la lista de nombres directamente en el código. No se pide al usuario — los datos están en el programa.
5
Función def — recibe dos parámetros
buscar(lista, elemento). Recibe ambos como parámetros. Solo retorna True o False — no imprime nada.
1
for — recorrer la lista
for item in lista. Cantidad conocida → for. Dentro, comparar cada elemento.
4
if + return True — salida inmediata
Dentro del for: if elemento == item: return True. El return sale de la función inmediatamente — no sigue recorriendo. Esto es más eficiente.
4
return False — fuera del for
return False va FUERA e INDENTADO AL MISMO NIVEL que el for — no dentro. Si el bucle terminó sin encontrar nada, recién retorna False.
2
print en main
Llamar buscar(), guardar el resultado en variable, e imprimir mensaje según sea True o False. El mensaje va en main, no en la función.
💡

Error clásico: poner el return False dentro del for. Si está dentro, retorna False en la primera iteración que no coincide, sin terminar de buscar. Debe estar fuera.

08
Traductor simple con diccionario
Intermedio

Crea un 7diccionario con 8 palabras español→inglés definido fuera de cualquier función. Crea una 5función traducir(palabra) que acceda al diccionario global y 4retorne la traducción o "Palabra no encontrada". 3El while True sigue pidiendo palabras 4hasta que el usuario escriba "salir".

Pasos — sin código
7
Diccionario global
Definir diccionario = {"hola": "hello", ...} al nivel del módulo — fuera de toda función. Al estar fuera, todas las funciones pueden accederlo sin necesidad de pasarlo como parámetro.
5
Función traducir(palabra)
Solo recibe la palabra a buscar. Accede al diccionario directamente porque es global. Retorna la traducción — no imprime.
8
Normalizar entrada
Dentro de la función, aplicar .lower().strip() antes de buscar. Así "Hola", "HOLA" y "hola" funcionan igual.
7
.get() con valor por defecto
return diccionario.get(palabra, "Palabra no encontrada"). Una sola línea que maneja el caso de clave existente y no existente.
3
while True — bucle de consultas
No sabemos cuántas palabras buscará → while True. Pedir palabra, verificar si es "salir", si no → llamar traducir() e imprimir.
4
Condición de salida — al inicio
if palabra == "salir": break. La condición de salida siempre va al inicio del while, antes de procesar el dato.
09
Lista de compras con presupuesto
Intermedio

Crea un programa que 3pida productos y precios uno por uno hasta que el usuario escriba "listo". 6Guarda cada par en un diccionario dentro de una lista. Al final 2muestra todos los productos, el total y 4si supera el presupuesto de $50.

Pasos — sin código
6
Lista vacía — inicializar
compras = [] antes del bucle. Todo acumulador va antes del bucle que lo llena.
3
while True — capturar productos
No sabemos cuántos productos habrá → while True. Pedir el nombre, verificar si es "listo" → break. Si no, pedir precio.
4
Condición de salida — al inicio
if entrada == "listo": break. Va al inicio del while antes de pedir el precio, para evitar pedir precio de "listo".
7
Diccionario por producto + append
Crear {"producto": nombre, "precio": float(precio)} y hacer compras.append(...). Cada producto es un diccionario. La lista los agrupa a todos.
8
Acumulador total
total = 0 antes del segundo bucle. Inicializar acumuladores justo antes del bucle que los usa.
1
for — calcular total y mostrar
for item in compras: total += item["precio"]. Ahora sí sabemos cuántos hay → for. Mostrar nombre y precio de cada uno.
4
if — comparar con presupuesto
if total > 50 → mensaje de advertencia. else → mensaje de aprobación. Va después de calcular el total, nunca dentro del bucle.
💡

Patrón clave: usa while True para capturar (no sabes cuántos). Luego usa for para procesar (ya sabes cuántos hay). Son dos bucles con propósitos distintos.

10
Conversor de unidades con tuplas
Intermedio

Crea una 6lista de tuplas (valor, unidad_origen). Crea una 5función convertir(valor, unidad) que convierta: 4km→millas, kg→libras, °C→°F, o "Unidad desconocida". 1Recorre la lista con for usando unpacking y 2muestra el resultado de cada conversión.

Pasos — sin código
6
Lista de tuplas — definir datos
Definir medidas = [(10, "km"), (5, "kg"), (100, "C"), ...] con al menos 6 tuplas usando las 3 unidades. Más de una por unidad para probar bien.
5
Función convertir(valor, unidad)
Recibe los dos valores desempacados. Solo calcula y retorna el resultado como string con unidad destino incluida. No imprime.
4
if/elif/else — una rama por unidad
if unidad == "km" → valor × 0.621. elif unidad == "kg" → valor × 2.205. elif unidad == "C" → (valor × 9/5) + 32. else → "Unidad desconocida".
5
return — resultado formateado
Retornar el resultado como string incluyendo la unidad destino: f"{valor} km = {resultado:.2f} millas". El formato va dentro de la función.
1
for con unpacking
for valor, unidad in medidas. El unpacking extrae ambos elementos de la tupla en una sola línea. Es la forma correcta — no uses índices.
2
print — en cada vuelta del for
Dentro del for: print(convertir(valor, unidad)). La función retorna el string completo — solo necesitas imprimirlo.
✅

Regla de la función: una función que calcula no imprime. Por eso convertir() retorna el string y el print() está en el for — no dentro de la función.
