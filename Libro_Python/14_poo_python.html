POO
¿Qué es un objeto?
clase · instancia · atributo · método

En POO todo gira alrededor de dos ideas: estado (qué datos tiene algo) y comportamiento (qué puede hacer). Un objeto agrupa ambas cosas. Una clase es el molde o plantilla; una instancia es el objeto concreto creado a partir de ese molde.

Sin POO — datos sueltos
# datos de un perro
nombre = "Rex"
raza   = "Labrador"
edad   = 3

# otro perro → más variables sueltas
nombre2 = "Luna"
raza2   = "Poodle"
edad2   = 5
# escala muy mal
Con POO — datos agrupados
class Perro:
    def __init__(self, nombre, raza, edad):
        self.nombre = nombre
        self.raza   = raza
        self.edad   = edad

rex  = Perro("Rex",  "Labrador", 3)
luna = Perro("Luna", "Poodle",   5)
# escala perfecto
Anatomía de una clase
classDefine el molde — el nombre de la clase va en PascalCase por convención
__init__Constructor — corre automáticamente cuando creas una instancia. Inicializa atributos.
selfReferencia al objeto actual. Siempre es el primer parámetro de cualquier método de instancia.
métodoFunción definida dentro de una clase. Tiene acceso a self y por tanto a todos los atributos.
dunderMétodos especiales con doble guión bajo: __str__, __len__, __add__. Python los llama automáticamente.
@propertyConvierte un método en atributo de solo lectura (o lectura/escritura) sin cambiar la sintaxis de uso.
@classmethodMétodo que recibe la clase en vez de la instancia. Útil para constructores alternativos.
@staticmethodFunción dentro de la clase que no recibe self ni cls. No necesita acceder al estado del objeto.

Atributos de clase vs atributos de instancia
class Perro:
    # atributo de CLASE — compartido por TODAS las instancias
    especie = "Canis lupus familiaris"
    contador = 0

    def __init__(self, nombre):
        # atributo de INSTANCIA — único por objeto
        self.nombre = nombre
        Perro.contador += 1      # modifica el atributo de clase

rex  = Perro("Rex")
luna = Perro("Luna")

print(rex.especie)     # Canis lupus familiaris — accede al de clase
print(luna.especie)    # mismo valor — compartido
print(Perro.contador) # 2 — cuántos perros se crearon

# si asignas a self.especie, crea un atributo de INSTANCIA que SOMBREA al de clase
rex.especie = "lobo"      # solo cambia en rex
print(rex.especie)     # lobo
print(luna.especie)    # Canis lupus familiaris — sin cambio
Regla de búsqueda de atributos: Python busca primero en el objeto (instancia), luego en la clase, luego en las clases padre. Si lo encuentra en el objeto, usa ese; si no, sube a la clase.
init
__init__ y self
constructor · inicialización · referencia al objeto

__init__ es el método que Python llama automáticamente al crear una instancia. No crea el objeto — eso lo hace __new__ — pero lo inicializa: asigna los valores iniciales de sus atributos. self es simplemente el objeto que se acaba de crear, pasado como primer argumento.

__init__ con validación
class CuentaBancaria:
    def __init__(self, titular, saldo_inicial=0):
        if saldo_inicial < 0:
            raise ValueError("El saldo inicial no puede ser negativo")
        self.titular = titular
        self.saldo   = saldo_inicial
        self.movimientos = []        # lista vacía — diferente por instancia

c1 = CuentaBancaria("Lian", 500)
c2 = CuentaBancaria("Ana")         # saldo_inicial=0 por defecto

print(c1.titular, c1.saldo)   # Lian 500
print(c2.titular, c2.saldo)   # Ana 0
self no es una palabra reservada — podrías llamarlo this o yo y funcionaría. Pero self es la convención universal en Python. No la rompas.
self siempre es el primer parámetro de cualquier método de instancia — Python lo pasa automáticamente cuando haces objeto.metodo().
Cómo Python traduce obj.metodo() a la llamada real
# esto que escribes tú:
c1.depositar(100)

# es exactamente igual a esto:
CuentaBancaria.depositar(c1, 100)
# Python inyecta el objeto como primer argumento (self)

Métodos de instancia
definir · llamar · acceder a self
Clase completa con métodos
class CuentaBancaria:
    def __init__(self, titular, saldo=0):
        self.titular     = titular
        self.saldo       = saldo
        self.movimientos = []

    def depositar(self, monto):
        if monto <= 0:
            raise ValueError("El monto debe ser positivo")
        self.saldo += monto
        self.movimientos.append(f"+{monto}")

    def retirar(self, monto):
        if monto > self.saldo:
            raise ValueError("Saldo insuficiente")
        self.saldo -= monto
        self.movimientos.append(f"-{monto}")

    def estado(self):
        print(f"Titular : {self.titular}")
        print(f"Saldo   : S/ {self.saldo:.2f}")
        print(f"Movs.   : {self.movimientos}")

# uso
cuenta = CuentaBancaria("Lian", 1000)
cuenta.depositar(500)
cuenta.retirar(200)
cuenta.estado()
# Titular : Lian
# Saldo   : S/ 1300.00
# Movs.   : ['+500', '-200']

Encapsulamiento
_ público · _ privado · __ name mangling

Encapsulamiento significa controlar qué partes internas de un objeto son accesibles desde afuera. Python no tiene modificadores como private o protected de Java — usa convenciones de nombres.

Convenciones de acceso en Python
class Ejemplo:
    def __init__(self):
        self.publico      = "accesible desde cualquier lugar"
        self._protegido   = "convención: no toques desde afuera"
        self.__privado    = "name mangling activo"

e = Ejemplo()
print(e.publico)          # ✓ funciona
print(e._protegido)      # ✓ funciona, pero no deberías
print(e.__privado)       # ✗ AttributeError
print(e._Ejemplo__privado) # ✓ así se llama realmente (name mangling)
Name mangling: cuando Python ve self.__nombre dentro de una clase Foo, lo renombra internamente a _Foo__nombre. Esto evita colisiones en herencia, no es verdadera privacidad. En Python "somos todos adultos" — la convención _ es suficiente señal de "no toques esto".
inh
Herencia
clase padre · clase hija · super() · override · MRO

Herencia permite que una clase hija reutilice el código de una clase padre, pudiendo extenderlo o modificarlo. La hija hereda todos los métodos y atributos del padre automáticamente.

Herencia básica
class Animal:                           # padre / base
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad   = edad

    def describir(self):
        print(f"{self.nombre}, {self.edad} años")

    def sonido(self):
        print("...")                        # comportamiento base genérico


class Perro(Animal):                      # hija — hereda de Animal
    def __init__(self, nombre, edad, raza):
        super().__init__(nombre, edad)    # llama al __init__ del padre
        self.raza = raza                   # atributo propio de Perro

    def sonido(self):                      # override — reemplaza el del padre
        print("¡Guau!")

    def info_raza(self):                   # método nuevo — solo en Perro
        print(f"Raza: {self.raza}")


class Gato(Animal):
    def sonido(self):
        print("¡Miau!")


# uso
rex  = Perro("Rex", 3, "Labrador")
michi = Gato("Michi", 2)

rex.describir()    # heredado de Animal → Rex, 3 años
rex.sonido()       # override en Perro  → ¡Guau!
rex.info_raza()    # propio de Perro    → Raza: Labrador
michi.sonido()    # override en Gato   → ¡Miau!

isinstance(rex, Perro)   # True
isinstance(rex, Animal)  # True — es instancia de la clase padre también
issubclass(Perro, Animal) # True
super() — extender en vez de reemplazar
class Animal:
    def describir(self):
        print(f"Animal: {self.nombre}")

class Perro(Animal):
    def describir(self):
        super().describir()         # llama el método del padre primero
        print(f"Raza: {self.raza}")  # luego agrega lo suyo

# resultado:
# Animal: Rex
# Raza: Labrador
Herencia múltiple y MRO
class A:
    def metodo(self): print("A")

class B(A):
    def metodo(self): print("B")

class C(A):
    def metodo(self): print("C")

class D(B, C):      # herencia múltiple
    pass

d = D()
d.metodo()          # → B  (MRO: D → B → C → A)

# ver el MRO completo
print(D.__mro__)
# (<class 'D'>, <class 'B'>, <class 'C'>, <class 'A'>, <class 'object'>)
MRO (Method Resolution Order): el orden en que Python busca un método en la cadena de herencia. Algoritmo C3 — busca de izquierda a derecha, depth-first, sin repetir. Toda clase hereda de object al final.

Implementar dunders en una clase Vector2D
class Vector2D:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        # legible para humanos
        return f"({self.x}, {self.y})"

    def __repr__(self):
        # reproducible — cómo recrear el objeto
        return f"Vector2D({self.x!r}, {self.y!r})"

    def __add__(self, otro):
        # habilita: v1 + v2
        return Vector2D(self.x + otro.x, self.y + otro.y)

    def __mul__(self, escalar):
        # habilita: v * 3
        return Vector2D(self.x * escalar, self.y * escalar)

    def __eq__(self, otro):
        # habilita: v1 == v2
        return self.x == otro.x and self.y == otro.y

    def __abs__(self):
        # habilita: abs(v) — magnitud del vector
        return (self.x**2 + self.y**2) ** 0.5

    def __bool__(self):
        # True si el vector no es el origen (0,0)
        return self.x != 0 or self.y != 0

# uso
v1 = Vector2D(3, 4)
v2 = Vector2D(1, 2)

print(v1)            # → (3, 4)         llama __str__
repr(v1)             # → Vector2D(3, 4) llama __repr__
print(v1 + v2)       # → (4, 6)         llama __add__
print(v1 * 3)        # → (9, 12)        llama __mul__
print(v1 == v2)      # → False          llama __eq__
print(abs(v1))       # → 5.0            llama __abs__ (3²+4²=25, √25=5)
print(bool(v1))      # → True           llama __bool__
__str__ vs __repr__:
__str__ → para humanos, legible. Lo usa print() y f-strings.
__repr__ → para desarrolladores, debe ser "reproducible" — idealmente eval(repr(obj)) == obj.
Si solo implementas uno, implementa __repr__ — Python lo usará como fallback para __str__.
@prop
@property
getter · setter · deleter · atributo controlado

@property convierte un método en un atributo que se accede sin paréntesis. Permite agregar lógica de validación al leer o escribir un atributo, sin cambiar cómo se usa desde afuera. El código externo no nota la diferencia.

Sin @property — el problema
class Persona:
    def __init__(self, edad):
        self.edad = edad       # atributo público sin control

p = Persona(25)
p.edad = -5                # ← edad negativa, Python no protesta
p.edad = "hola"            # ← string como edad, tampoco
Con @property — getter y setter
class Persona:
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad   = edad        # esto llama al setter automáticamente

    @property
    def edad(self):               # GETTER — se llama al leer p.edad
        return self._edad

    @edad.setter
    def edad(self, valor):        # SETTER — se llama al escribir p.edad = x
        if not isinstance(valor, int):
            raise TypeError("La edad debe ser un entero")
        if valor < 0 or valor > 130:
            raise ValueError(f"Edad inválida: {valor}")
        self._edad = valor         # guarda en _edad (privado por conv.)

    @edad.deleter
    def edad(self):               # DELETER — se llama al hacer del p.edad
        print("Eliminando edad")
        del self._edad

    @property
    def es_mayor(self):           # property de solo lectura (sin setter)
        return self._edad >= 18

# uso — idéntico al de un atributo normal
p = Persona("Lian", 22)
print(p.edad)          # → 22      (llama getter, sin paréntesis)
p.edad = 23            # llama setter, valida
p.edad = -5            # → ValueError: Edad inválida: -5
print(p.es_mayor)      # → True    (property calculada, sin paréntesis)
del p.edad             # llama deleter
La magia de @property: el código que usa la clase no cambia — sigue escribiendo p.edad = 23, no p.set_edad(23). Esto respeta la interfaz pública mientras agrega validación internamente. Es la forma Pythónica de hacer getters/setters.
@cls
@classmethod y @staticmethod
métodos alternativos · métodos de utilidad · cls vs self
Tres tipos de métodos en una clase:
· Método de instancia — recibe self. Accede al objeto y a la clase. El más común.
· @classmethod — recibe cls (la clase, no el objeto). No accede a instancias individuales.
· @staticmethod — no recibe self ni cls. Es una función normal dentro de la clase por organización.
@classmethod — constructores alternativos (uso más común)
class Fecha:
    def __init__(self, dia, mes, año):
        self.dia = dia
        self.mes = mes
        self.año = año

    @classmethod
    def desde_string(cls, texto):
        # constructor alternativo — crea una Fecha desde "DD-MM-AAAA"
        dia, mes, año = [int(n) for n in texto.split("-")]
        return cls(dia, mes, año)   # cls() == Fecha() — funciona en subclases

    @classmethod
    def hoy(cls):
        import datetime
        h = datetime.date.today()
        return cls(h.day, h.month, h.year)

    def __str__(self):
        return f"{self.dia:02d}/{self.mes:02d}/{self.año}"

# tres formas de crear una Fecha
f1 = Fecha(12, 4, 2026)              # constructor normal
f2 = Fecha.desde_string("12-04-2026") # classmethod
f3 = Fecha.hoy()                      # classmethod
print(f1, f2, f3)   # 12/04/2026  12/04/2026  (hoy)
@staticmethod — función de utilidad relacionada con la clase
class Temperatura:
    def __init__(self, celsius):
        self.celsius = celsius

    @staticmethod
    def celsius_a_fahrenheit(c):
        # no necesita self ni cls — es una conversión pura
        return c * 9/5 + 32

    @staticmethod
    def es_caluroso(celsius):
        return celsius > 30

    def en_fahrenheit(self):
        # método de instancia que usa el staticmethod internamente
        return Temperatura.celsius_a_fahrenheit(self.celsius)

# se puede llamar desde la clase o desde una instancia
print(Temperatura.celsius_a_fahrenheit(100))  # → 212.0

t = Temperatura(37)
print(t.en_fahrenheit())                       # → 98.6
print(Temperatura.es_caluroso(37))             # → True
Cuándo usar cada uno — resumen visual
class MiClase:

    def metodo_instancia(self):
        # ✓ necesito acceder a self.atributo
        # ✓ necesito modificar el estado del objeto
        pass

    @classmethod
    def metodo_clase(cls):
        # ✓ necesito acceder a atributos de la CLASE (no del objeto)
        # ✓ quiero crear instancias con lógica diferente (constructor alternativo)
        # ✓ quiero que funcione bien en subclases (cls en vez de hardcodear la clase)
        pass

    @staticmethod
    def funcion_utilitaria():
        # ✓ no necesito self ni cls
        # ✓ la función pertenece conceptualmente a la clase pero es independiente
        # ✓ podría ser una función suelta, pero la pongo aquí por organización
        pass
Ejemplo completo — todo junto en una clase real
class Producto:
    _igv = 0.18                             # atributo de clase — IGV Perú

    def __init__(self, nombre, precio):
        self.nombre = nombre
        self.precio = precio                 # usa el setter

    @property
    def precio(self):
        return self._precio

    @precio.setter
    def precio(self, valor):
        if valor < 0:
            raise ValueError("Precio no puede ser negativo")
        self._precio = valor

    @property
    def precio_con_igv(self):              # property calculada
        return self._precio * (1 + Producto._igv)

    @classmethod
    def cambiar_igv(cls, nuevo_igv):       # modifica atributo de clase
        cls._igv = nuevo_igv

    @classmethod
    def desde_dict(cls, datos):           # constructor alternativo
        return cls(datos["nombre"], datos["precio"])

    @staticmethod
    def es_precio_valido(precio):         # utilidad sin estado
        return isinstance(precio, (int, float)) and precio >= 0

    def __str__(self):
        return f"{self.nombre}: S/ {self.precio:.2f} (+ IGV: S/ {self.precio_con_igv:.2f})"

    def __repr__(self):
        return f"Producto({self.nombre!r}, {self.precio!r})"

    def __lt__(self, otro):               # habilita sorted() y comparaciones
        return self._precio < otro._precio

# uso completo
p1 = Producto("Laptop", 3500)
p2 = Producto.desde_dict({"nombre": "Mouse", "precio": 80})

print(p1)                      # Laptop: S/ 3500.00 (+ IGV: S/ 4130.00)
print(p1 < p2)                 # False (3500 no es menor que 80)
print(Producto.es_precio_valido(-10))  # False

productos = [Producto("C",300), Producto("A",100), Producto("B",200)]
print(sorted(productos))       # ordena por precio usando __lt__
