List comprehension básica
for simple · transformación · equivalencia con bucle

Una list comprehension es un for comprimido que construye una lista. Lee izquierda a derecha: "dame expresión para cada item en iterable".

Con for + append
cuadrados = []
for n in range(1, 6):
    cuadrados.append(n ** 2)
# [1, 4, 9, 16, 25]
Con comprehension
cuadrados = [n ** 2 for n in range(1, 6)]
# [1, 4, 9, 16, 25]

# mismo resultado, una sola línea
Ejemplos básicos
# transformar strings
palabras  = ["hola", "mundo", "python"]
mayusculas = [p.upper() for p in palabras]
# ['HOLA', 'MUNDO', 'PYTHON']

# longitud de cada palabra
longitudes = [len(p) for p in palabras]
# [4, 5, 6]

# aplicar función propia
def doble(x): return x * 2
[doble(n) for n in range(5)]
# [0, 2, 4, 6, 8]

# iterar sobre string (cada carácter)
[c.upper() for c in "python"]
# ['P', 'Y', 'T', 'H', 'O', 'N']

# convertir tipos
[int(x) for x in ["1", "2", "3"]]
# [1, 2, 3]

Con filtro — if al final
filtrar elementos · condición de inclusión

Agrega un if al final para incluir solo los elementos que cumplan la condición. El elemento que no cumple simplemente se descarta — no se agrega nada.

Filtrar con if
numeros = [1, 2, 3, 4, 5, 6, 7, 8]

# solo pares
[n for n in numeros if n % 2 == 0]
# [2, 4, 6, 8]

# solo positivos
[n for n in [-3, 1, -1, 4, 0] if n > 0]
# [1, 4]

# strings que empiezan con 'a'
frutas = ["manzana", "banana", "arándano", "pera"]
[f for f in frutas if f.startswith("a")]
# ['arándano']

# filtrar y transformar a la vez
[n ** 2 for n in range(10) if n % 2 == 0]
# cuadrados de los pares del 0 al 9
# [0, 4, 16, 36, 64]
if/else dentro de la expresión — ternario
# if/else en la EXPRESIÓN (parte izquierda) — transforma según condición
# formato: valor_si_true if condición else valor_si_false

["par" if n % 2 == 0 else "impar" for n in range(5)]
# ['par', 'impar', 'par', 'impar', 'par']

# reemplazar negativos por 0
[n if n >= 0 else 0 for n in [-2, 3, -1, 5]]
# [0, 3, 0, 5]
if al final (filtro) vs if/else en la expresión:
[x for x in lista if condicion] → filtra, puede devolver lista más corta
[a if cond else b for x in lista] → transforma, siempre devuelve mismo largo
El if/else en la expresión no puede estar solo — necesita el else.

Comprehensions anidadas
for dentro de for · aplanar listas · matrices
Regla de lectura: los for en una comprehension se leen de izquierda a derecha, igual que si los escribieras anidados. El primer for es el bucle externo, el segundo es el interno.
Dos for — equivalencia con bucles anidados
# bucles anidados equivalentes
resultado = []
for i in [1, 2, 3]:
    for j in ["a", "b"]:
        resultado.append((i, j))
# [(1,'a'),(1,'b'),(2,'a'),(2,'b'),(3,'a'),(3,'b')]

# comprehension equivalente
[(i, j) for i in [1, 2, 3] for j in ["a", "b"]]
# mismo resultado
Aplanar lista de listas
matriz = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]

# aplanar — un solo for sobre cada sublista, otro sobre cada elemento
[n for fila in matriz for n in fila]
# [1, 2, 3, 4, 5, 6, 7, 8, 9]

# aplanar con filtro
[n for fila in matriz for n in fila if n % 2 == 0]
# [2, 4, 6, 8]
Comprensión en la expresión — lista de listas (transponer)
# expresión que es otra comprehension → lista de listas
matriz = [[1, 2, 3],
           [4, 5, 6]]

# transponer (filas → columnas)
[[fila[i] for fila in matriz] for i in range(3)]
# [[1, 4], [2, 5], [3, 6]]
Cuidado con el abuso: una comprehension con 2 for y un if ya es difícil de leer. Si necesitas 3 for o lógica compleja, usa bucles normales — la legibilidad gana sobre la brevedad.

Dict y set comprehensions
{ } en vez de [ ] · pares clave:valor · valores únicos
Dict comprehension — { clave: valor for ... }
# cuadrados como dict {n: n²}
{n: n**2 for n in range(1, 6)}
# {1:1, 2:4, 3:9, 4:16, 5:25}

# invertir un dict (clave ↔ valor)
original = {"a": 1, "b": 2, "c": 3}
{v: k for k, v in original.items()}
# {1:'a', 2:'b', 3:'c'}

# filtrar un dict — solo pares con valor > 1
{k: v for k, v in original.items() if v > 1}
# {'b': 2, 'c': 3}

# crear dict desde dos listas con zip
nombres = ["Ana", "Luis", "Mía"]
notas   = [15, 18, 12]
{n: nota for n, nota in zip(nombres, notas)}
# {'Ana': 15, 'Luis': 18, 'Mía': 12}
Set comprehension — { expresión for ... }
# set de cuadrados — sin duplicados automáticamente
{n**2 for n in [1, 2, 2, 3, 3]}
# {1, 4, 9}   — duplicados eliminados

# primeras letras únicas de una lista de palabras
palabras = ["manzana", "mango", "pera", "piña"]
{p[0] for p in palabras}
# {'m', 'p'}

# diferencia: set {} vs dict {} — si no hay : es set, si hay : es dict
{x for x in lista}      # set comprehension
{x: 0 for x in lista}   # dict comprehension
{}                        # dict vacío — para set vacío usa set()

Generator expressions
( ) en vez de [ ] · lazy · sin construir la lista completa

Cambiar [] por () crea un generador en vez de una lista. La diferencia: la lista calcula todos los valores de inmediato y los guarda en memoria. El generador calcula cada valor solo cuando se le pide — perfecto para datos grandes.

Lista vs generador
# lista — calcula todo YA, guarda en memoria
lista = [n**2 for n in range(1000000)]   # ocupa ~8MB

# generador — calcula uno a uno cuando se necesita
gen = (n**2 for n in range(1000000))      # ocupa ~100 bytes

# se puede usar en funciones que esperan un iterable
sum(n**2 for n in range(10))              # → 285  (sin [] extra)
max(len(p) for p in ["hola", "python"])    # → 6
any(n < 0 for n in [1, 2, -3, 4])          # → True
all(n > 0 for n in [1, 2, 3])              # → True

¿Cuándo usarlas?
legibilidad · límites · reglas prácticas
✓ Úsalas cuando:
· La lógica cabe en una línea legible
· Transformas o filtras un iterable
· Quieres un dict/set nuevo a partir de otro
· Pasas el resultado a sum/max/any/all
✗ Evítalas cuando:
· Necesitas más de 2 for anidados
· Tienes lógica compleja (try/except, múltiples if)
· Necesitas efectos secundarios (prints, logs)
· La línea supera ~80 caracteres
La guía de oro: si tienes que explicar verbalmente qué hace la comprehension, probablemente debería ser un bucle normal. La claridad siempre gana.
