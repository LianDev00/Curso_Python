01
Validar contraseña con lista de errores
función retorna lista · main imprime cada error
✓
Crea una función validar(password) que revise 3 reglas:
· Mínimo 8 caracteres
· No puede tener espacios
· Debe tener al menos un dígito

La función debe retornar una lista con los mensajes de error que apliquen.
Si no hay errores, retorna una lista vacía.

En el cuerpo principal, pide la contraseña, llama la función y:
· Si la lista está vacía → imprime "✓ Contraseña válida"
· Si tiene errores → imprime cada uno con un for
Lo que entrena
La función no imprime nada — solo acumula errores con .append() y los retorna.
Todos los print viven en el cuerpo principal.
La trampa: querer usar return dentro de cada if para salir rápido — eso solo muestra el primer error.
02
Calculadora con función separada
función pura · input/output solo en main
✓
Crea una función calcular(a, b, operacion) que reciba dos números y un string ("suma", "resta", "multiplicacion", "division") y retorne el resultado.
Si la operación es "division" y b == 0, retorna None.

En el cuerpo principal, pide los dos números y la operación al usuario.
Si el resultado es None, imprime "Error: división por cero".
Si no, imprime el resultado con 2 decimales.
Lo que entrena
La función no hace input() ni print().
Retorna None como señal de error — el main decide qué mostrar.
Usar if resultado is None en el main para manejar ese caso.
03
Función que analiza una lista de notas
función retorna múltiples valores · main los usa
✓
Dada la lista [15, 8, 12, 20, 9, 17, 11], crea una función analizar(notas) que retorne:
· El promedio
· La nota más alta
· Cuántos aprobaron (nota ≥ 11)

La función retorna los tres valores juntos.
En el cuerpo principal, llama la función, desempaca los tres valores e imprímelos.
Lo que entrena
Una función puede retornar múltiples valores con: return promedio, maximo, aprobados
En el main se desempaca: prom, maximo, aprobados = analizar(notas)
La función no imprime. Solo calcula y retorna.
04
Función es_palindromo limpia
función retorna bool · mensaje en main · responsabilidades claras
✓
Crea una función es_palindromo(texto) que:
· Convierta a minúsculas y elimine espacios
· Retorne True o False

En el cuerpo principal, pide una palabra, llama la función y muestra:
· "{palabra}" ES palíndromo ✓
· "{palabra}" NO es palíndromo ✗

Pruébala con: "Ana", "reconocer", "anita lava la tina", "python"
Lo que entrena
La función retorna solo True/False — no el mensaje.
El mensaje y el f-string van en el if/else del main.
La limpieza del texto ocurre dentro de la función, no en el main.
Trampa clásica
Retornar (True, "Es palíndromo") desde la función. Si lo haces, la función está asumiendo responsabilidades que no le corresponden.

· 1 hora → S/ 3.00

· 6 o más horas → S/ 12.00
· 0 o negativo → "Tiempo inválido"

10
Validador de año bisiesto
condiciones anidadas · orden importa · casos límite reales
✓
Un año es bisiesto si:
· Es divisible por 4
· Excepto si es divisible por 100 — en ese caso no es bisiesto
· Excepto si también es divisible por 400 — en ese caso sí es bisiesto

Pide un año e imprime si es bisiesto o no.
Lo que entrena
Las excepciones tienen orden: primero el caso más específico (÷400), luego el general (÷100), luego ÷4.
Prueba con: 2000 (bisiesto ÷400), 1900 (NO bisiesto ÷100), 2024 (bisiesto ÷4), 2023 (no bisiesto).
Si inviertes el orden de los elif, el resultado cambia.
Nivel de dificultad
Este ejercicio es el más difícil del nivel 2. Las condiciones se encadenan con lógica AND/OR. Si te trabas, pide solo una pista — no la solución completa.
