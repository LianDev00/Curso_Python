Preguntas que debes hacerte antes de escribir código
¿Necesito repetir algo?
→ bucle
Si sí: ¿sé cuántas veces? → for. ¿No sé? → while
¿Necesito tomar una decisión?
→ if / elif / else
¿Una condición? → if/else. ¿Varias? → elif
¿Necesito guardar varios valores?
→ lista o diccionario
¿Con nombre? → dict. ¿Solo valores? → list
¿Voy a reutilizar esta lógica?
→ función def
Si lo vas a llamar más de una vez, es una función
¿Necesito un contador o suma?
→ variable acumuladora
Inicializa en 0 antes del bucle, modifica dentro
¿El usuario puede meter datos inválidos?
→ try / except
Siempre que conviertas input a int o float
01
Analizador de texto
Intermedio

Crea un programa que 8pida al usuario un texto. Luego, 1recorre cada carácter del texto y 4clasifícalo como vocal, consonante, número o símbolo. 6Guarda los resultados en cuatro listas separadas. Al finalizar, 2muestra cuántos hay de cada tipo y 2cuál es el carácter más frecuente del texto.

Pasos — sin código
8
Variable / Input
Pedir el texto al usuario con input() y guardarlo en una variable. Normalizar con .lower().strip() para no distinguir mayúsculas.
6
Listas — inicializar
Crear cuatro listas vacías antes del bucle: vocales, consonantes, numeros, simbolos. Los acumuladores siempre van ANTES del bucle.
1
Bucle for
Usar for caracter in texto. Sabemos cuántos caracteres hay (la longitud del string), así que for es la elección correcta — no while.
4
Condicional if / elif / else
Dentro del bucle, clasificar cada carácter con if/elif/else: ¿está en "aeiou"? → vocal. ¿es .isalpha()? → consonante. ¿es .isdigit()? → número. Si no → símbolo.
6
append() — acumular
En cada rama del if, hacer .append(caracter) a la lista correspondiente.
7
Diccionario — contar frecuencias
Para encontrar el carácter más frecuente, usar el patrón conteo[c] = conteo.get(c, 0) + 1 dentro del bucle. Al salir, usar max(conteo, key=conteo.get).
2
print() — mostrar resultados
Fuera del bucle, imprimir len() de cada lista y el carácter más frecuente. Todo el output va DESPUÉS de procesar, nunca dentro del bucle clasificador.
¿Por qué estas herramientas?
¿Repetir?
Sí → for (longitud conocida)
¿Decidir?
Sí → if/elif/else (4 casos)
¿Guardar múltiples?
Sí → 4 listas separadas
¿Contar por clave?
Sí → diccionario
02
Sistema de votación
Intermedio

Crea un 5programa con una función que simule un sistema de votación. 3El programa sigue pidiendo votos hasta que el usuario escriba "fin". Cada voto es un nombre de candidato. 7Guarda los votos en un diccionario donde la clave es el candidato y el valor es la cantidad de votos. 4Si el candidato no existe en el diccionario, agrégalo. Al terminar, 2muestra los resultados ordenados de mayor a menor y 2declara al ganador.

Pasos — sin código
5
Función def
Separar en dos funciones: registrar_votos() que maneja el bucle de captura, y mostrar_resultados(votos) que muestra el reporte. El diccionario pasa como argumento.
7
Diccionario — inicializar
Crear votos = {} antes del bucle. Los diccionarios son la estructura correcta cuando necesitas relacionar una clave (nombre) con un valor (conteo).
3
Bucle while True
No sabemos cuántos votos habrá — el usuario decide cuándo parar. Eso significa while True con un break cuando detectes "fin". Nunca uses for aquí.
4
Condicional — condición de salida
Lo primero dentro del while: if voto == "fin": break. La condición de salida siempre va al inicio del bucle, antes de procesar el dato.
7
Diccionario — patrón get()
Usar votos[candidato] = votos.get(candidato, 0) + 1. Esto agrega el candidato si no existe (con 0) y suma 1. No necesitas if para verificar si existe.
2
print() — resultados ordenados
Fuera del bucle, ordenar con sorted(votos.items(), key=lambda x: x[1], reverse=True) y recorrer con for para imprimir. El ganador es el primero de la lista ordenada.
💡

Clave mental: cuando el enunciado dice "hasta que el usuario escriba algo" o "hasta que ocurra una condición", siempre es while True + break. El for es para cuando sabes cuántas vueltas.

03
Inventario de tienda
Avanzado

Crea un sistema de inventario con un 7diccionario de productos donde cada producto tiene nombre, precio y stock. El sistema tiene un 3menú que se repite hasta que el usuario elija salir. Las opciones son: ver inventario, 4buscar producto y mostrar si existe o no, y registrar una venta 4validando que haya suficiente stock. Cada opción debe ser una 5función separada. 1Al ver el inventario, recorre todos los productos y 2muéstralos en formato tabla. Al final de cada venta, 8actualiza el stock del producto.

Pasos — sin código
7
Diccionario anidado — estructura de datos
Lo primero es definir la estructura: inventario = {"producto": {"precio": x, "stock": y}}. Diseña la estructura antes de escribir cualquier función.
5
Funciones def — una por responsabilidad
Identificar las 3 funciones del menú: ver_inventario(inv), buscar_producto(inv, nombre), registrar_venta(inv). Cada función hace UNA sola cosa.
3
while True — menú principal
El menú se repite indefinidamente — while True. Dentro, pedir la opción y llamar a la función correspondiente. La opción "salir" ejecuta break.
1
for — recorrer inventario
Para ver el inventario: for nombre, datos in inventario.items(). Sabemos cuántos productos hay — el for es correcto. Usar f-strings con alineación para simular tabla.
4
if — validaciones
Dos validaciones en registrar_venta: if nombre not in inventario → "no existe". if cantidad > inventario[nombre]["stock"] → "stock insuficiente". Solo si pasa ambas → proceder.
8
Variable — actualizar stock
Después de validar: inventario[nombre]["stock"] -= cantidad. El operador -= es la forma corta de restar y guardar en la misma variable.
2
print() — output formateado
Usar f"{nombre:<15} S/{precio:>6.2f} {stock:>6}" para alinear columnas. El :<15 alinea a la izquierda con 15 caracteres de ancho.
✅

Patrón mental clave: cuando el enunciado dice "cada opción debe ser una función", primero lista las opciones del menú y crea una función por cada una. Luego el while solo llama funciones — nunca hace lógica directamente.

04
Analizador de notas con estadísticas
Avanzado

Crea un programa donde 3el usuario ingresa notas una por una hasta escribir "listo". 6Guarda todas las notas en una lista. Luego, usando 5funciones separadas, calcula: 8promedio, 1nota más alta y más baja, y 4clasifica cada nota como Aprobado (≥60) o Reprobado (<60). 2Muestra un reporte final con todos estos datos y 4si el promedio es ≥70, muestra un mensaje de felicitación.

Pasos — sin código
6
Lista — inicializar
Crear notas = [] antes del bucle. Todo lo que se acumula va en una lista vacía que se llena dentro del bucle.
3
while True — captura
No sabemos cuántas notas habrá → while True. Pedir input, verificar si es "listo" → break. Si no, convertir a float y hacer .append() a la lista.
5
Funciones def — separar cálculos
Una función por cálculo: calcular_promedio(notas), clasificar(notas). Reciben la lista como parámetro y retornan el resultado. No imprimen — solo calculan.
1
for + max/min
Para la nota más alta y más baja: usar directamente max(notas) y min(notas). Para clasificar, recorrer la lista con for nota in notas y usar if/else.
4
if/else — clasificar y felicitar
Dos condicionales independientes: uno para nota >= 60 (dentro del for), otro para promedio >= 70 (fuera del for, en el reporte final).
2
print() — reporte final
Todo el output va después de llamar las funciones. Llama cada función, guarda en variable, luego imprime. Nunca mezcles cálculo con output en la misma línea.
💡

Regla de oro: una función que calcula no imprime. Una función que imprime no calcula. Separar estas dos responsabilidades hace el código limpio y reutilizable.

05
Gestor de contactos con búsqueda
Complejo

Crea un gestor de contactos donde 7cada contacto es un diccionario con nombre, teléfono y email, y 6todos los contactos se guardan en una lista. El sistema tiene un 3menú que se repite indefinidamente con cuatro opciones, 5cada una como función: agregar contacto, 1listar todos los contactos recorriendo la lista, 1buscar por nombre recorriendo la lista y 4comparando si el nombre coincide, y eliminar contacto 4verificando primero que exista. 2Si la lista está vacía, avisa al usuario antes de intentar listar o buscar.

Pasos — sin código
6
Lista de diccionarios — estructura
La estructura es contactos = [] donde cada elemento es {"nombre": x, "tel": y, "email": z}. Lista porque hay muchos contactos. Diccionario porque cada uno tiene campos con nombre.
5
Funciones def — mapear el menú
Antes de escribir código, listar las 4 funciones: agregar(contactos), listar(contactos), buscar(contactos, nombre), eliminar(contactos, nombre). La lista siempre va como parámetro.
3
while True — menú principal
El menú repite sin condición de fin predefinida → while True. Dentro: mostrar opciones, pedir opción, llamar función. La opción "4" o "salir" ejecuta break.
4
if — guardia antes de procesar
En listar y buscar: if len(contactos) == 0: print("Lista vacía") return. El return sale de la función inmediatamente. Esta es la "guardia" — siempre va al inicio.
1
for — recorrer lista
Para listar: for c in contactos: print(c["nombre"], c["tel"]). Para buscar: for c in contactos: if c["nombre"].lower() == busqueda.lower(). El for es correcto porque sabemos cuántos hay.
4
if — validar antes de eliminar
Para eliminar: primero buscar si existe con el for. Si encontró, usar .remove(). Si no encontró, avisar. No intentes eliminar sin verificar primero — causa errores.
2
print() — feedback claro
Después de cada operación, confirmar qué pasó: "Contacto agregado", "Contacto no encontrado", "Contacto eliminado". El usuario siempre debe saber si su acción funcionó.
Mapa mental del problema
¿Muchos contactos?
→ lista principal
¿Cada contacto con campos?
→ diccionario dentro
¿Menú infinito?
→ while True
¿4 acciones distintas?
→ 4 funciones
¿Recorrer contactos?
→ for (cantidad conocida)
¿Validar existencia?
→ if + guardia al inicio
